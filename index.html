博客改版

2017年01月07日
好久没更新过博客了，决定重新再出发，对博客进行了改变，以后多多更新。 博客原来的内容基本没删除（只所以说基本，是因为我忘了事前备份，而且直接使用的push -f把之前github上的内容全覆盖了），不管之前写的内容多糟糕，想法看法对不对，还是留着当个纪念吧！ 之所以好久没更新，除了这半年多家里出了些事情外，还有就是因为找不到更新的意义。我对这个博客的定位是一个技术博客，但大部分技术文章只是记录，缺少自己的看法和想法，另外目前我所可能遇到的技术问题基本都能在网上找到解决方法，或者我就是通过网上的方法解决的，缺乏原创性内容。我认为博客不应该是一些记录性的内容，博客主要还是给其他人看的，要不自己做笔记就可以了，个人也不应该通过看博客的记录性内容学习新的知识，缺乏系统性。 现在想法有点变化，对记录性的内容没有这么排斥了，有些记录性的内容可以为自己服务，容易找到，有些问题解决方案虽然网上有，但再次遇到重新搜索总是麻烦的。针对缺少原创性内容的问题，可以通过将记录性内容转化为总结来解决，不一味的记录详细内容，而是偏向总结供人参考。还有重要的一点，博客不仅仅是给其他人看的，还记录了自己的成长！ 这次对博客进行改版，不仅仅是换了主题，而且换了静态博客生成工具，将博客从jekyll迁移到了Hugo。 Hugo是使用go语言编写的静态网站生成工具，得益于go语言的特性，hugo可以在大部分平台上运行，而且不需要安装依赖。我放弃jekyll的一个很大原因是我平时不用ruby，为了博客却要安装一大堆相关的ruby环境软件，而hugo只需要一个程序就可（好吧，我早就把jekyll环境给卸了，重新安装比较麻烦，但最后发现建新主题更麻烦，::>_<:: ）。 另外博客主题是使用青砾的博客改造而来，未征得博主的同意，但毕竟是自己将其前端页面改造成了主题，有一些新增的内容，中间有自己的劳动，就擅自使用了，非常抱歉。另基于以上原因，主题暂时不放到github上了！ 下一步计划将rss订阅功能添加到博客中！

年俗

2016年02月24日
我家住在鲁西南地区的一个小村庄里，可能是因为离孔子老家比较近，各种过年的风俗习惯和需要遵守的规矩比较多。最近几年年轻一辈由于偷懒都不太注意遵守各种风俗习惯了，这些风俗习惯或者说规矩是迷信也好、传统也罢，可能已经不太适应这个飞速发展的社会，也可能早已忘记这些风俗背后的意义内涵，但至少是祖祖辈辈传下来的，感觉有必要记录一下。需要说明的是，家乡这边各个村庄的年俗有细微差别，比如我姥爷家，骑自行车过去只需10多分钟，但他们那过年一些细节和我家就不太一样。 首先需要说明的是，我们庄两三百户左右人家都姓刘，追根溯源都是一个祖先，但时间长了以后血脉关系自然又分远近，于是分成了一个个大的家族（我们这叫“门”，俗称“一门子人”就是指一个家族的，血脉关系比较近），每个家族都有一份家谱，整个村庄有总的家谱。另外下面的叙述中可能不自觉的夹杂着家乡话的语音习惯，敬请原谅！ 下面按时间(阴历)顺序叙述我们这的年俗，大年三十前的各种准备没有严格的顺序限制，可以按需安排，除夕和大年初一一些活动不能乱。ps：由于今年重感冒，有些内容没留下照片，用的上一年的图，另外也没有作为主力参与过年的准备活动，可惜！ 年前主要是做各种食物为年后招待亲戚做准备，我们这年后初六前集市商店一般是不开门卖东西的，所以各种蔬菜肉食年前要买好准备好。由于地处北方，冬季严寒，各种食物能在自然环境下放比较长的时间。 腊月二十二 腊月二十二今年是蒸的菜馒头，也叫菜馍，就是菜包子。旧时大家家里都穷，全是菜馅，现在虽然加肉，但菜馒头的称呼也延续下来了。蒸菜馒头时我正好发烧，没有拍照片，贴张上年的。 自己做的菜馒头和卖的包子差不多，主要差别是自己做的个头大，外形没有包子店的外形好。馅料传统的是红萝卜和白萝卜加粉条，今年自己院子里种的平菇正好那几天长势好，吃不完，就弄了些蘑菇馅。由于自己吃而且要一直吃到年后正月初十左右，所以做的量比较大，一般用家里的土灶每锅两篦能做3锅左右。之所以先做菜馒头是因为有陷大家喜欢吃，先做了吃着。 腊月二十四 蒸花糕（又叫枣花）和馒头，花糕和馒头如下图，不是馒头的都叫花糕。 第三个图应该叫花杉(音是这个，具体哪个字不好说)。还有其他的花样没有拍全或者不会做。可惜由于发烧的原因没参与，我可是主力呜呜。。，上面最后一张图是我上年拍的，我自己做的。第四个图中是土灶用的大铁锅，刚换了一个，虽经过打磨处理，仍可以看出使用时还是会产生铁锈，补铁，吼吼！ 馒头就是普通的馒头，有些时候会包些豆馅馒头，豆馅用红豆和枣加糖熬制。最近几年由于家人不爱吃，过年时只是象征性的做几个，外形和馒头一样。 蒸馒头和花糕由于量大比较费时费力，一般要花费一整天或者更多，酵母要提前一天发酵，由于要吃到年后正月初十左右(我们这过年后正月十五前不能蒸馒头，有风俗要留一块花糕到二月二龙抬头吃)，一般需要蒸4-5锅两篦的花糕和馒头，烧锅就要半天多。 腊月二十五 炖肉 过年时家家都要买二三十斤猪肉，取其中带皮的切成大方块炖了，切成方块炖熟容易存放，吃时拿出一块切下几片肉片就可以了。另需留下些纯瘦肉冻起来年后招待客人，还要分出一部分炸酥肉和做肉丸子。 其他还需买些猪杂、猪头肉等做好的熟食留着年后用。 我们这没有做腊肉的习惯，可能有个别人家会做，但大部分人家不会。 腊月二十七 腊月二十七由于白天有其他的事情，在傍晚的时候做了些油炸的食物，炸的酥肉、丸子和鱼。由于我被同学喊出去玩了，没在家参与，回来拍的照。 酥肉是面里面加上瘦肉丝油炸而成的长条状食物。由于人多，需要的量大，我家做总是面多肉少，面里面裹些瘦肉，有些人家做是瘦肉上面挂一些面糊炸的，基本全是肉。 突然发现丸子忘了拍照了，应该是放在酥肉下面了，但是没拍到。 丸子是用白菜帮切粒混合葱姜加到面粉里挤成球形油炸而成的。上面图中依稀看到的圆形就是丸子，和酥肉差不多颜色。丸子炸完现吃比较好吃。 另外还炸了一些带鱼和鲤鱼，油炸的原因是容易存放，油炸后不容易腐败。 其他事项 前面准备食物的阶段各个食物准备的顺序不重要，可以打乱，一般不容易存放的东西比如煮肉晚一些，容易存放的食物比如蒸馒头做早一些。前面时间有间隔是因为其他一些事情如打算庭院，收拾东西等比较琐碎不方便细说。 另外我们这有一个风俗就是yuan yu，家乡话，不清楚是哪两个字，意思是为过年做的食物刚做好时需要先敬神才能吃。取出一点食物样品，嘴里喊着各路神仙的名字，请他们先享用，然后往天上抛洒，弄完后就可以自己吃了，有个经典的话是老灶爷就锅吃，一类食物只需一次就行，不用每个人吃前都yuan yu一次。 除夕那天主要内容就不是准备食物了，就开始过年了，其中杂七杂八的风俗规矩比较多。 腊月二十九(除夕) 今年由于没有大年三十，所以腊月二十九就是除夕。除夕开始规矩就比较多了，不像前面主要是准备食物。 首先早晨不能喝粥（我们这边粥一般指的玉米粥，和南方不太一样，清汤米饭不叫粥），说法是喝粥糊涂一年。吃完早饭后煮浆糊贴对联，一般我爸和我弟贴对联，我这做饭小能手和我妈及奶奶一起包饺子，下面一篦的饺子全是我包的，很饱满吧，哈哈。这里说个小技巧，包饺子和面时加点盐和鸡蛋清面会比较劲道，饺子下锅后不容易烂，不要加鸡蛋黄。 全是我包的 大年三十和初一的饺子不能乱包，要统一。正月十五包饺子可以各种花样玩。 下面是贴的对联，刚补拍的，第一张是堂屋，第二张是大门，大门已经20年左右了，也没翻修过，比较破。今年对联没找熟人写，全买的，都是卖给做生意人的对联，没仔细挑。 包完饺子就快接近中午了，下面的活动一定要在十二点之前做完。 首先是一边下水饺一边给家里敬的神和祖先烧香和烧元宝。家里有画像和瓷人的神一共五个，不包括贴的门神。每个神都要烧一炷香，另外需注意的是先给老天爷烧香，然后是观音菩萨和灶王爷，接着是关爷财神和仙姑奶奶，这里强调一下是因为灶王爷在厨房，而其他神都在堂屋，不注意可能就先给堂屋所有神仙烧了。另外烧香时还要烧元宝，一般元宝烧6个、8个或者多少都行，六六大顺，8就是发。我妈过年一烧就烧十六个。元宝都是手工用薄黄纸叠的，如下图，忘记拍个特写了，算上大年初一早晨烧的要叠好多好多！ 与此同时，还要在每个门的门两旁各烧一根香和一个元宝，屋里的门不需要，邻院的门才烧。另外有个规矩是折断的香一定不能用。 烧完香和元宝就可以饺子出锅了，首先也要先yuan yu，让各路神仙和祖先先品尝，这里yuan yu需要用碗盛着饺子汤洒在神像和祖先牌位前。 yuan yu以后就可以点鞭炮了，然后大家一起吃水饺。还有一个习俗是饺子做好后给长辈送一碗，奶奶都是在我家这边过年，大伯一般会让堂哥送一碗过来，然后爸妈让堂哥带一碗我家的饺子回去。由于今年二堂哥孩子出生不满一岁，担心家里寒冷，就没回家过年，大伯亲自来的。今年大伯饺子送的早，我家饺子还没熟，等饺子熟了，我又端一碗跑了一趟大伯家。 还有一个事情是家里敬完神去土地庙里和家谱(家谱一个家族一个，放在一个人家里)上炷香和烧元宝（一般烧香和元宝都是配合的）。 下面放几张家里供的神的照片，除夕和大年初一烧香真的是烟熏火燎，睁不开眼。 老天爷 观音菩萨，开过光的，每年腊月二十六我妈都要摆贡(我们这边的说法，就是摆贡品)给其过生日，然后换衣服。 关公和财神 仙姑奶奶 灶王爷，上面有二十四节气的阴历时间。 还有门神是贴对联时贴在门上的。 秦叔宝和尉迟恭，还有其他门神就不贴了。 上午活动一般在十二点前完成。 下午事情就不多了，主要是去家里坟地烧香请祖先回家过年，然后包饺子及其他活动为初一早晨准备。傍晚时，还做了些肉丸子，这个也是过年必备的。 晚上我家是没有年夜饭的（这里应该有个哭的表情），主要还是为大年初一做准备。大年初一规矩比较多，有些东西需要晚上睡前准备好。因为这些准备内容涉及大年初一的风俗，我把这部分放到大年初一来说。 大年初一 先说下需要提前准备的内容。 首先，大年初一早晨需要为神仙和祖先摆贡(摆放贡品敬神)，因为早晨比较忙，需要提前准备好，一般需要煮些白菜叶、洗些香菜放在贡品上，贡品就是前面介绍的含肉的食物，另外的贡品就是前面蒸的枣花。一般老天爷和观音菩萨贡品多些（六种），其他神少些。神那里要放两根香作为筷子。 还有就是因为初一早晨需要点的蜡烛和香比较多，需要先准备好，尤其是门旁点蜡烛不方便，需要准备个容器。 其次，需要在进出家里的大门和院门下放一截很粗的树干(俗称棍子)挡着，这个习俗啥意思不太懂。再就是在院子里撒上芝麻杆(就是芝麻的植物茎晒干)，上面有芝麻的壳，踩着可啪可啪的响，据我妈说是因为姜子牙的老婆怕响，这样做就不敢进门了。 大年初一早晨要早起，凌晨4、5点钟左右起床。拜年等活动在要天亮前结束。以下规矩一般在天亮前有效。 先提一下，起床后不能出去上厕所，要尿在屋内的小便盆里(农村厕所都是单独的，冬天冷，都用尿壶或者小便盆），洗脸用的水也不能倒外面。 项目需要迁移到linux环境，准确的说是redhat系统，所以最近研究在redhat下java调用matlab的解决方法，期间遇到了各种问题，卡了好长时间，现在终于解决了。 Redhat下matlab安装 首先需要在redhat下安装matlab，我redhat是搭在虚拟机里的，大家也可以在虚拟机里搭建练手，可以保存快照，很方便回退。 这里不详细介绍安装过程，只提一些重点内容。 安装时提示是否在线安装选否(询问是英文的) 报错：error while loading shared libraries: libXp.so.6: cannot open shared object file: No such file or directory时，将安装文件中/update/bin/glnx86目录下的libmwins.so文件复制到/usr/lib/下，命名为libXp.so.6 安装就这两个需要注意的地方，下面是调用 java调用matlab matlab导出jar包这里就不提了，大家可以自己搜索。这里主要讲述过程中遇到的困难及解决办法 提示MCR版本不对，这个报错很奇怪，明明一个matlab导出的，用的就是它的MCR，居然提示不对，解决方法就是配置环境变量LD_LIBRARY_PATH,XAPPLRESDIR LD_LIBRARY_PATH=/usr/local/MATLAB/bin/glnxa64 XAPPLRESDIR=/usr/local/MATLAB/X11/app-defaults LD_PATH=/usr/local/MATLAB/runtime/glnxa64 将'/usr/local/MATLAB/'换成自己的安装目录,如果还有问题，在eclipse里面配置运行的环境变量，方法是右键项目,选择Run As下Run Configurations,点击Environment选项卡，新建上面两个环境变量。 报错Exception in thread "main" java.lang.LinkageError:/usr/lib/libXp.so.6:undefined symbol :SSL_connect，这里是缺少文件，解决方法是： mounts /dev/cdrom /media （挂载盘） cd /media && ls （查看是否挂载成功） vim /etc/yum.repos.d/rhel-debuginfo.repo 打开该文件 *****该文件中前两行不需要改，从baseurl开始修改成下文所示的值 [xxxx] name=xxxx baseurl=file:///media/Server enable=1 （这里一定是1，启动服务） gpgchek=0 （这里一定是0，否则下面不能安装） 保存该文件，执行下面操作就ok yum -y install *linbXp* （安装需要的包） rpm -qa libXp (查看是否安装成功） 然后就可以java调用matlab了。 友情提示：上面对rhel-debuginfo.

我对分类的一些理解

2014年07月02日
最近一直在做阿里的大数据竞赛，对数据挖掘中的分类问题有了一些自己的理解，在这里总结记录一下。 特征 分类问题中有一个很重要的概念是特征，特征代表了训练集中数据的各种属性。各种分类器模型以特征为参数进行训练，特征是分类问题中的关键。 在比赛的进行过程中，我逐渐领悟到特征才是分类问题中的核心。特征的选取要保证既能有效区别于其他类别的样本，又和所属类别的样本尽量近似或保持一致，所以特征不是越多越好，也不是越少越好，而是越能代表不同类别的差别越好。如果一个特征在所有类别的样本中是一样的数值，肯定不能使用它来分类，而一个特征在所有类别样本中无规律或者说近似随机，也不能有效的分类。特征的选取直接决定了使用算法或模型对数据进行分类的能达到的极限。 模型 前面说明的特征的重要性，那各种算法各种模型是干什么的呢？我现在的理解就是模型是为了学习特征，从特征的差别中区分类别。这里没说很明白，再解释一下。假设我有一个二类问题，我提取出样本的一个特征，我认为这个特征有用，即能区别与其他类别，又和所属类别高度统一。我把这个特征加入到模型的训练集中去，我不会想当然的认为这个新提取的特征值越高数据越有可能属于正类或负类，而是交给模型判断，让模型去学习，模型说这个特征值越高越有可能是正类，那就认为确实是这样，或者我不管是不是这样，交给模型去学习去，学出啥是啥。模型学的不好可能是模型本身的问题，如逻辑回归是线性的，不能判断一个特征值低中高三段两头类别一样中间不一样的情况，也可能是特征没选好，不能有效区分不同类别的差异。这里就弱化了模型的作用，模型就是从特征中学习出类别的差异，或者推理出类别的差异，重点是特征能代表这种差异，而模型的选择主要是看其学习能力和推理能力。 分类预测 提取特征、选择模型训练后就要对数据进行分类预测，而有些分类问题为什么不能做到很好的正确率和召回率？我认为主要有以下原因： 1.问题本身有很大的随机性或不可预测性 像这次阿里的大数据竞赛，通过用户4个月的购买行为预测第5个月的用户会购买的品牌。一个用户的购买行为本身具有很大的随机性(当然也有一定的规律性)，用户在第5个月的购买行为受太多随机因素的影响，比如你看到别人穿的一件衣服感觉很好看也想购买，或者你看到别人穿的一件自己正打算买的衣服，由于撞衫决定不买了。这些问题本身的随机性导致不能准确预测。 2.数据包含的信息有限 还是以阿里比赛为例吧，数据中只包含了品牌(注意不是商品)的id，没有其他关于品牌的任何信息如类别等,这些数据包含的信息有限，如果有品牌的类别信息，比如知道这个品牌是大家电，前4个月买了以后第五个月一般不会再次购买，如果还要买的话也要考虑换个品牌买了。通过这些有限的信息很难准确区分不同类别的差异。 3.经验技术问题 这个不是针对问题本身了，如果你对某一类分类问题有很多的经验和技术积累，你就可能得到较好的效果，没有则效果就可能差些。
